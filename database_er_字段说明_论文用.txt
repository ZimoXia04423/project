================================================================================
数据库 E-R 与五张表字段说明（论文正文可删减后粘贴）
与项目 src/database.py 中 MySQL 实现一致，数据库名：fake_review_detection

【图注建议】
图 X-X  虚假评论识别系统数据库 E-R 关系图（5 张表）
图中实体矩形内为主要字段摘要；各字段含义见下文说明。评论主表通过外键分别关联商品信息表与用户信息表；爬取任务日志表与数据清洗日志表记录批次级任务与统计信息，用于流程追溯与审计，实现中未与单条评论建立外键关联。

--------------------------------------------------------------------------------
（1）商品信息表 products
--------------------------------------------------------------------------------

| 字段名（逻辑） | 类型（示意） | 说明 |
| --- | --- | --- |
| id | INT AUTO | 内部物理主键，自增 |
| product_id | VARCHAR(32) | 商品唯一编号，非空、唯一，业务侧主键 |
| product_name | VARCHAR(256) | 商品名称 |
| category | VARCHAR(64) | 商品类目 |
| shop_name | VARCHAR(128) | 店铺名称 |
| price | DECIMAL(10,2) | 商品价格 |
| total_sales | INT | 总销量，默认 0 |
| platform | VARCHAR(16) | 所属电商平台 |
| created_at | TIMESTAMP | 记录创建时间 |

--------------------------------------------------------------------------------
（2）用户信息表 users
--------------------------------------------------------------------------------

| 字段名（逻辑） | 类型（示意） | 说明 |
| --- | --- | --- |
| id | INT AUTO | 内部物理主键，自增 |
| user_id | VARCHAR(64) | 用户唯一编号，非空、唯一，业务侧主键 |
| nickname | VARCHAR(64) | 用户昵称 |
| register_days | INT | 注册天数，默认 0 |
| total_reviews | INT | 历史评论总数，默认 0 |
| avg_rating | DECIMAL(3,2) | 平均评分 |
| credit_level | INT | 信用等级（如 1–5），默认 0 |
| platform | VARCHAR(16) | 所属平台 |
| created_at | TIMESTAMP | 记录创建时间 |

--------------------------------------------------------------------------------
（3）评论主表 reviews（核心事实表）
--------------------------------------------------------------------------------

| 字段名（逻辑） | 类型（示意） | 说明 |
| --- | --- | --- |
| id | INT AUTO | 内部物理主键，自增 |
| review_id | VARCHAR(64) | 评论唯一编号，非空、唯一 |
| product_id | VARCHAR(32) | 关联商品 ID，外键 → products(product_id) |
| user_id | VARCHAR(64) | 关联用户 ID，外键 → users(user_id) |
| review_text | TEXT | 评论正文，非空 |
| rating | INT | 评分（如 1–5） |
| review_time | VARCHAR(32) | 评论时间（字符串存储） |
| platform | VARCHAR(16) | 来源平台 |
| crawl_time | VARCHAR(32) | 数据采集时间 |
| review_length | INT | 评论字数（特征） |
| exclamation_count | INT | 感叹号数量（特征） |
| repeat_char_ratio | DOUBLE | 重复字符比例（特征） |
| sentiment_score | DOUBLE | 情感得分（特征，如 0–1） |
| user_review_count | INT | 用户历史评论数（特征） |
| avg_review_interval_days | DOUBLE | 平均评论间隔天数（特征） |
| label | INT | 标签：-1 未标注，0 真实，1 虚假 |
| is_cleaned | TINYINT | 是否已完成清洗流程标记 |
| is_duplicate | TINYINT | 是否重复评论标记 |
| created_at | TIMESTAMP | 入库时间 |

--------------------------------------------------------------------------------
（4）爬取任务日志表 crawl_logs
--------------------------------------------------------------------------------

| 字段名（逻辑） | 类型（示意） | 说明 |
| --- | --- | --- |
| id | INT AUTO | 主键 |
| task_name | VARCHAR(64) | 任务名称 |
| platform | VARCHAR(16) | 目标平台 |
| target_products | INT | 目标商品数量 |
| total_requests | INT | 总请求数 |
| success_count | INT | 成功请求数 |
| fail_count | INT | 失败请求数 |
| total_reviews | INT | 获取评论条数 |
| start_time | VARCHAR(32) | 开始时间 |
| end_time | VARCHAR(32) | 结束时间 |
| status | VARCHAR(16) | 任务状态，如 completed |
| created_at | TIMESTAMP | 记录时间 |

--------------------------------------------------------------------------------
（5）数据清洗日志表 cleaning_logs
--------------------------------------------------------------------------------

| 字段名（逻辑） | 类型（示意） | 说明 |
| --- | --- | --- |
| id | INT AUTO | 主键 |
| total_before | INT | 清洗前评论总数 |
| duplicates_removed | INT | 去重删除条数 |
| empty_removed | INT | 空白删除条数 |
| short_removed | INT | 过短删除条数 |
| total_after | INT | 清洗后剩余条数 |
| cleaning_time | VARCHAR(32) | 清洗耗时描述 |
| created_at | TIMESTAMP | 记录时间 |

================================================================================
 cardinality 表述建议（正文一句话）
================================================================================
一件商品可对应多条评论（1:n）；一名用户可发表多条评论（1:n）；评论主表通过
product_id、user_id 分别参照商品表与用户表，满足参照完整性。爬取日志与清洗日志
按任务批次记录统计信息，便于与业务流程对齐，不与单条评论强制建立外键。
