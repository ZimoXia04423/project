# 骨折分级智能检测系统 - 系统架构设计文档

## 1. 系统总体架构

系统采用四层架构设计，自上而下分别为：

### 1.1 表现层 (GUI Layer)
- **位置**: `gui/`
- **职责**: 基于 PyQt5 实现图形界面，向用户提供操作入口
- **主要组件**:
  - `MainWindow`: 主窗口界面
  - `DetectionThread`: 异步检测线程
  - 图像显示组件、控制面板、结果展示组件

### 1.2 服务层 (Service Layer)
- **位置**: `service/`
- **职责**: 负责模型加载、图像预处理、结果后处理和文件保存
- **主要服务**:
  - `ModelService`: 模型管理服务（加载、切换、卸载）
  - `ImageService`: 图像处理服务（加载、预处理、缩放）
  - `ResultService`: 结果分析服务（骨折分级判定、报告生成）
  - `FileService`: 文件保存服务（图像、文本、目录管理）

### 1.3 算法层 (Algorithm Layer)
- **位置**: `models/`, `preprocess/`
- **职责**: 负责改进 YOLOv10 模型的训练、验证和推理
- **主要模块**:
  - `models/`: YOLO 模型配置与注意力机制定义
  - `preprocess/`: 图像预处理与数据增强

### 1.4 数据层 (Data Layer)
- **位置**: `datasets/`
- **职责**: 负责原始图像、标签文件、训练集与测试集管理
- **主要组件**:
  - `YOLODataset`: YOLO 格式数据集加载
  - `DatasetManager`: 数据集管理器

## 4. 核心模块说明

### 4.1 图像预处理模块 (`preprocess/image_preprocess.py`)

**功能**:
1. 灰度归一化：统一像素值范围 [0, 255]
2. CLAHE 增强：使用对比度受限自适应直方图均衡化提升骨皮质边界清晰度
3. 尺寸标准化：统一缩放至模型所需输入尺寸（如 640×640）
4. 噪声抑制：采用中值滤波减少随机噪声对纹理提取的干扰

**使用示例**:
```python
from preprocess import ImagePreprocessor

preprocessor = ImagePreprocessor(
    target_size=(640, 640),
    clip_limit=2.0,
    use_denoise=True
)

processed_image = preprocessor.preprocess(original_image)
```

### 4.2 数据增强模块 (`preprocess/data_augmentation.py`)

**功能**:
1. 随机翻转与轻微旋转：增强对体位变化的鲁棒性
2. 亮度与对比度扰动：增强对曝光差异的适应性
3. 随机缩放与平移：提升模型对位置变化的泛化能力
4. Mosaic 增强：可选，需谨慎使用（避免破坏医学结构）
5. MixUp 增强：可选，需谨慎使用

**使用示例**:
```python
from preprocess import DataAugmenter

augmenter = DataAugmenter(
    h_flip_prob=0.5,
    rotation_range=(-15, 15),
    use_mosaic=False
)

augmented_image, augmented_boxes = augmenter.augment(image, boxes)
```

### 4.3 骨折分级器 (`service/result_service.py`)

**判定规则**:
1. **重度骨折**: 检测到 fracture + periostealreaction
2. **中度骨折**: 检测到 fracture + (softtissue 或 pronatorsign)
3. **轻度骨折**: 仅检测到 fracture
4. **未见骨折**: 未检测到 fracture

## 5. 扩展性设计

系统预留了以下扩展空间：

1. **批量检测**: 在 `infer/` 模块中已实现 `batch_inference` 函数
2. **检测历史记录**: 通过 `FileService` 可实现结果持久化
3. **模型性能对比**: `ModelService` 支持多模型加载与切换
4. **网页端展示**: 服务层可独立为 API，供 Web 前端调用
5. **移动端部署**: 模型可导出为 ONNX/TorchScript 格式

## 6. 使用方式

### 6.1 训练模型
```python
from train import Trainer

trainer = Trainer(
    data_yaml='yolo_dataset_final/data.yaml',
    model_type='yolov10',
    epochs=100,
    batch_size=16
)

trainer.train()
```

### 6.2 批量推理
```python
from infer import batch_inference

results = batch_inference(
    image_dir='test_images/',
    weights_path='weights/yolov10_best.pt',
    output_dir='results/'
)
```

### 6.3 启动 GUI
```bash
python gui/__init__.py
```

或运行原主程序：
```bash
python main.py
```

## 7. 技术栈

- **深度学习框架**: PyTorch, Ultralytics YOLO
- **GUI 框架**: PyQt5
- **图像处理**: OpenCV, NumPy
- **数据格式**: YOLO 格式（TXT 标签）
- **模型**: YOLOv8, YOLOv10

## 8. 注意事项

1. 医学图像增强需谨慎，避免破坏解剖结构语义
2. 垂直翻转到医学图像可能不适用（器官位置会颠倒）
3. Mosaic 和 MixUp 增强在医学图像上效果需验证
4. 模型权重文件较大，建议使用 GPU 训练和推理
